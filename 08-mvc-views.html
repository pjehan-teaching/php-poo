Base Controller

Nous allons maintenant créer une classe PHP qui nous servira de base à nos controllers :

// vendor/MyCompany/Controller/Controller.php
namespace MyCompany\Controller;

abstract class Controller
{
    protected string $viewPath;
    protected string $template;

    protected function render($view, $variables = []): void
    {
        ob_start(); // Start buffering the output
        extract($variables); // Convert array to variables
        require($this->viewPath . $view . '.php');
        $content = ob_get_clean(); // Store the output in a variable
        require($this->viewPath . $this->template . '.php');
    }
}

App Controller

Nous allons maintenant créer une classe PHP qui nous servira de base à nos controllers mais cette fois-ci spécifiquement pour notre application :

// src/Controller/AppController.php
namespace App\Controller;

use MyCompany\Controller\Controller;

abstract class AppController extends Controller
{
    public function __construct()
    {
        $this->viewPath = __DIR__ . "/../Views/";
        $this->template = "base";
    }
}

HomeController

À présent, nous allons modifier le controller HomeController.

Dans un premier temps, nous allons créer un template de page :

<!-- src/Views/home/index.php -->
<?php
// Déclaration des variables pour la vue
/** @var $username string */
?>
<h1>Homepage</h1>
<?= $username; ?>

Puis dans le controller HomeController :

// src/Controller/HomeController.php
namespace App\Controller;

class HomeController extends AppController
{
    public function index(): void
    {
        $this->render("home/index", ["username" => "johndoe"]);
    }
}

MVC

Cette étape nous permet d'améliorer notre architecture MVC en séparant clairement la logique de l'application (controllers) de la présentation (vues) tout en utilisant un template commun pour l'ensemble des pages.
